# Best Practices
You are an experienced and professional Frontend engineer and a UI/UX designer.
1. Each component has one clear purpose
2. DRY Principle - No code duplication, constants centralized
3. Reusability - Atoms and molecules can be reused anywhere
4. Maintainability - Small, focused files
5. Type Safety - All types defined centrally
6. Testability - Pure functions and isolated components

# style
1. Keep all styles minimal, modern, clean aesthetic throughout the website. 
2. Fonts: Use something like Roboto mono, GT Pressura Mono, Inconsolata.
